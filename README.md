# 射手座之日 原型版

这是一个浏览器运行的《射手座之日》即时舰队对战原型。

## 代码结构（已合并维护）

- `shared/game-core.js`：统一游戏规则核心（舰船运动、分离、技能、视野、AI、战斗结算）。
- `src/game.js`：本地 AI 模式前端，仅负责输入与渲染，规则由 `shared/game-core.js` 驱动。
- `server/server.js`：网络对战服务端，仅负责房间/同步/网络协议，规则同样由 `shared/game-core.js` 驱动。
- `src/online-client.js`：网络大厅与对战客户端渲染/插值/输入发送。

后续若要调整平衡或技能机制，优先修改 `shared/game-core.js`，本地与联机会同时生效。

## 运行方式

### 1. 安装依赖（仅网络对战需要）

```bash
cd /Users/haruhi/haruhi-game
npm install
```

### 2. 启动静态页面服务

```bash
cd /Users/haruhi/haruhi-game
npm run start:web
```

### 3. 启动网络对战服务器（可选）

```bash
cd /Users/haruhi/haruhi-game
npm run start:server
```

### 4. 打开页面

- 本地 AI 对战：`http://localhost:8080/index.html`
- 网络对战大厅：`http://localhost:8080/online.html`

## 已实现机制

- 2D 实时对战，地图采用 3x3 九宫格战区。
- 主舰 + 两艘副舰的编队模式，共享编队血量逻辑。
- 一级分离与二级分离：
  - 一级分离：副舰一独立。
  - 二级分离：副舰二独立。
  - 每次分离后总属性约衰减到 80%（连续两次约为 64%）。
- 能量与速度联动：
  - 低推进更容易积累能量。
  - 高推进会降低回能并可能出现净耗能。
- 视野与侦察：
  - 仅显示当前视野内的敌方单位。
  - 敌方离开视野后不保留可见信息。
- 侦查机机制：
  - 只能指定派往某个战区。
  - 先直飞战区，再随机巡航。
  - 单次受击即毁，侦查机互撞同归于尽。
- 自动攻击机制：
  - 舰船与僚机自动攻击最近可见目标。
  - 具有弹道飞行与散布，可能落空。
  - 侧弦角度可获得更高输出效率。
- 舰长技能：
  - 凉宫春日（主舰）：向战区派遣战斗僚机。
  - 朝比奈实玖瑠（副舰二）：二级分离后可发射远程光束。
  - 长门有希（副舰一）：一级分离后视野扩大。
- 敌方 AI：
  - 自动机动、分离、侦查、技能释放。
- 网络对战大厅：
  - 可选择创建公开房、私人房、AI训练房。
  - 私人房支持随机 6 位房间号加入。
  - 可加入公开房间进行玩家对战。
  - 采用服务器权威固定 Tick（30Hz）与快照同步（15Hz）。
  - 客户端使用 Ping/Jitter 自适应插值缓冲，降低高延迟抖动观感。
  - 输入序号确认（Ack）机制，减少输入乱序造成的状态误差。

## 操作说明

- 选择舰船：在侧栏“操作舰船”下拉框中切换主舰/副舰。
- 战区选择：左键单击地图，选择对应战区。
- 航行指令：左键双击地图设置目标点，自动生成可视化贝塞尔航线。
- 航线调整：拖拽黄色控制点调整弧度，拖拽绿色终点调整落点；系统会按当前航向、速度与转向能力自动限制为可行曲线。
- 推进档位：通过“推进功率”滑块独立控制，不再依赖拖拽手势。
- 清除航线：使用“清除当前舰船航线”按钮恢复直航。
- 战区行动：在侧栏选择战区后派出侦查机或僚机。
- 实玖瑠光束：点击技能按钮后，再点击地图方向开火。

## 网络大厅操作说明

- 连接服务器：页面加载后自动连接 `ws(s)://当前网页域名:21246`，无需手动输入地址；“重新连接”按钮可手动重连。
- 创建房间：
  - “创建公开房”会出现在公开房列表中。
  - “创建私人房”会显示 6 位房间号。
  - “创建 AI 训练房”可直接开局。
- 加入房间：
  - 在公开房列表点击“加入”。
  - 或输入 6 位房间号加入私人房。
- 对战控制：
  - 左键单击：选择战区（本地标记）。
  - 左键双击：为当前舰船设定目标点。
  - 拖拽控制点/终点：在线调整贝塞尔曲线。
  - 推进功率滑块独立控制，可实时同步至服务器。
